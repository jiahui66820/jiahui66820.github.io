CSS 变量

CSS变量解决的问题

构建大型站点时，CSS的数量是非常庞大的。例如配色方案会在项目中大量、重复使用，后期维护会面对可维护性的挑战。

为了解决这个问题，比如可以采用less、sass等预处理器。

但是CSS变量具有天然的优势。不同于预处理器，必须最终编译成普通的CSS代码，CSS变量可以被浏览器识别解析，甚至可以用javascript访问或重新赋值。

w3c规范中这样描述CSS变量

使用CSS变量，给看似随机的值加上富有信息的名字，从而使得大文件更容易阅读和编辑，更少出错。因为，你只需要在自定义属性中改变一次值，所有应用了这个变量的地方都会自动跟着一起改变。

总结:

  1. 提高大型系统文档的可维护性
  2. 提高系统代码的可读性
  3. 可以在浏览器中实时修改生效
  4. JavaScript中可以取到CSS变量

CSS变量语法

  1. CSS变量的声明

    在变量前加上破折号，--

    但是变量名只能由普通字符组成，不能包含$，[]，^，%等特殊字符。普通字符局限在数字[0-9]，字母[a-zA-Z]，下划线_和短横线-的组合。

    变量名可以为中文、日文和韩文

  • 变量声明中，属性值不能直接进行数学运算，可以借助calc()函数进行运算

    ⚠️：不能在一个没有单位的值后直接加上单位，正确的拼接方式是借用calc()函数，进行计算

JS就这样沦为工具人

过去点击提示，点击切换等效果都需要JS针对特定的元素进行样式设置，现在有了CSS变量，我们只需要一段通用的全局JS就可以了，然后JS玩耍自己应该玩耍的，其他效果由CSS处理。

可以看到JavaScript再也不用负责任何与交互行为相关的逻辑，直接成为工具人，一个单纯的传递点击位置坐标，以及点击元素偏移和尺寸信息的工具人。
